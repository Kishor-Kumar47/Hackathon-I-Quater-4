# Data Model: Embedding Pipeline

## Document Chunk
**Description**: Represents a segment of text extracted from a Docusaurus URL

**Fields**:
- `id` (string): Unique identifier for the chunk
- `content` (string): The actual text content
- `source_url` (string): The URL from which the content was extracted
- `metadata` (dict): Additional information about the chunk
  - `title` (string): Page title
  - `section` (string): Section or heading under which content appears
  - `created_at` (datetime): Timestamp when chunk was created

## Embedding Vector
**Description**: Numerical representation of text content generated by Cohere

**Fields**:
- `id` (string): Unique identifier matching the document chunk
- `vector` (list[float]): The embedding vector values
- `source_chunk_id` (string): Reference to the original document chunk
- `model` (string): The Cohere model used to generate the embedding

## Processing Job
**Description**: Represents an extraction, embedding, and storage workflow

**Fields**:
- `id` (string): Unique identifier for the processing job
- `status` (string): Current status (pending, processing, completed, failed)
- `source_url` (string): The base URL being processed
- `total_documents` (int): Number of documents to process
- `processed_documents` (int): Number of documents processed
- `created_at` (datetime): When the job was started
- `completed_at` (datetime): When the job was completed (if applicable)
- `error_message` (string): Error details if job failed

## Qdrant Point
**Description**: Data structure for storing in Qdrant vector database

**Fields**:
- `id` (string): Unique identifier
- `vector` (list[float]): The embedding vector
- `payload` (dict): Metadata associated with the vector
  - `content` (string): Original text content
  - `source_url` (string): URL where content was found
  - `title` (string): Page title
  - `section` (string): Section where content appeared
